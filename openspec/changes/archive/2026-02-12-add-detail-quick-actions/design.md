## Context

当前应用在详情页（单品和搭配）缺乏直接的操作入口，导致用户在进行删除或查看大图时需要额外的点击步骤。

## Goals / Non-Goals

**Goals:**
- 在详情页提供快捷删除功能。
- 实现图片点击全屏预览。
- 保持 UI 的极简风格。

**Non-Goals:**
- 不涉及批量删除功能。
- 不涉及图片的编辑或裁剪功能。

## Decisions

- **删除按钮位置**: 在详情页右上角使用垃圾桶图标（或文字按钮），点击后弹出二次确认弹窗。
- **图片预览实现**: 使用现有的 UI 组件库（如 shadcn/ui 的 Dialog 或专门的 Lightbox 库）来实现全屏预览。
- **交互反馈**: 删除成功后使用全局 Toast 提示。

## Risks / Trade-offs

- **误删风险**: [Risk] → [Mitigation] 强制要求二次确认弹窗。
- **移动端适配**: [Risk] → [Mitigation] 确保全屏预览在不同屏幕尺寸下表现良好，支持手势关闭。
